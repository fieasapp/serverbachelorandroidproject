General

Start the socket server by pressing on the start button. Upon the server start a socket server will bet created listening to a port specified on the server settings. To stop the socket server simply press the stop button. This will close all clients connection and free the resources used on the process.


Before you start the server

  1. Make sure that the MQTT Broker is running. For instruction on how to run the MQTT Broker click here or go to the section below.
  2. Check that all settings are set and that the values are correct. Information on setting up server settings is in the next chapter below.

About Server Settings

It is very important that all information set in the server settings are correct to insure a proper running socket server. The option to edit and save server settings are available on the menu bar on top of th e main window or by hit the following shortcut keys "Ctrl+Alt+S" on your keyboard. Additional information are available on each settings by clicking on the info button next to the field[right side of the text field].

Information on the following settings will be explain one by one.

  • Server name
  • Server Port
  • Broker Address
  • Broker Port
  • Options file path
  • External Program path

Server name - A non functional dependant option. Provide a name for the server or use the local machine name by checking the check box under the text field.

Server Port - Enter a port number to be used by the socket server. The default port number is 1883. To find out which port number is assigned to an application or currently being used by other running processes, use the netstat function in the command window(on Windows OS only). For instruction click here.

Broker Address - Provide the Machine's IP-address where the MQTT Broker currently installed. It is important that the right address(IP) is provided for the application to work properly. If the broker is installed on the same machine as this application an easier functionality is provided to locate the machine's IP-address by checking the check box under the text field. Alternatively, typing "localhost" will be acceptable given that the MQTT broker reside locally.

Broker Port - Enter a port number being used by the MQTT broker. The default port number is 1883. To find out which port number is assigned to an application or currently being used by other running processes, use the netstat function in the command window(on Windows OS only). For instruction click here.

Options file path - the location of the "option.txt"-file used by the 3D motion tracking system. For more convenient used, browse through folders by using the search function to locate the file by clicking on the button next to the text field. This option is located on the "Files and Directories"-tab on the Server Settings options.

External Program path - the location of the executable-file used to run the 3D motion tracking system application. For more convenient used, browse through folders by using the search function to locate the file by clicking on the button next to the text field. This option is located on the "Files and Directories"-tab on the Server Settings options.
